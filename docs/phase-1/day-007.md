# Day 007 — The Pipeline

## Overview

The pipeline is the single most important concept in PowerShell. Everything else builds on it. If Day 1 was "hello world," Day 7 is where PowerShell actually starts to feel like a superpower.

The idea: the output of one command becomes the input of the next. You chain commands together with the `|` character (pipe). Instead of writing a script to process data step by step, you build a chain — each step passing objects to the next.

And remember — PowerShell passes **objects**, not text. That's what makes it more powerful than shell pipelines in bash.

---

## Your first pipeline

```powershell
# Get all running processes, sorted by memory usage
Get-Process | Sort-Object WorkingSet -Descending

# Get the top 5 memory-consuming processes
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 5

# Get all services and filter to only running ones
Get-Service | Where-Object { $_.Status -eq "Running" }
```

---

## The $_ variable

Inside a pipeline filter, `$_` represents the **current object** being processed. It's how you reach into each item as it passes through.

```powershell
# For each service, print its name if it's running
Get-Service | Where-Object { $_.Status -eq "Running" } | ForEach-Object { $_.Name }
```

---

## Chaining it together

```powershell
# Find the top 3 CPU-hungry processes and show just name and CPU
Get-Process |
    Sort-Object CPU -Descending |
    Select-Object -First 3 |
    Select-Object Name, CPU
```

Breaking a pipeline across multiple lines with `|` at the end makes complex chains readable. You'll do this constantly in real scripts.

---

## What you learned

- `|` passes objects from one cmdlet to the next
- PowerShell passes objects, not text — that's what makes it powerful
- `$_` is the current object in a pipeline
- `Where-Object`, `Sort-Object`, `Select-Object`, `ForEach-Object` are the core pipeline cmdlets
