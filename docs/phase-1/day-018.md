# Day 018 — Running Your First .ps1 Script

## Overview

Everything up to now has been typed directly into the terminal. That works for one-liners, but real scripts live in `.ps1` files. Today you save your first script to disk and actually run it — and you'll hit PowerShell's execution policy, which trips up almost every beginner the first time.

---

## Execution policy — what it is and why it exists

PowerShell has a security feature called execution policy that controls what scripts are allowed to run on a machine. By default on most Windows systems it's set to `Restricted` — meaning no scripts run at all.

```powershell
# Check your current policy
Get-ExecutionPolicy

# Check policy for all scopes
Get-ExecutionPolicy -List
```

Common policy levels:

| Policy | What it means |
|--------|--------------|
| `Restricted` | No scripts run (default on Windows clients) |
| `AllSigned` | Only scripts signed by a trusted publisher |
| `RemoteSigned` | Local scripts run; downloaded scripts need a signature |
| `Unrestricted` | All scripts run (not recommended for production) |
| `Bypass` | Nothing blocked — used in automation pipelines |

---

## Setting execution policy for your session

```powershell
# Set for current user only (recommended for dev work)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

This is the standard setting for a developer or admin workstation. It lets your local scripts run without requiring a signature, while still protecting against unsigned downloaded scripts.

---

## Creating and running your first script

1. Open VS Code (or Notepad), create a file called `hello.ps1`
2. Add this content:

```powershell
<#
.SYNOPSIS
    My first PowerShell script file.
#>

$name = Read-Host "What's your name?"
Write-Host "Welcome to PowerShell, $name!" -ForegroundColor Cyan
Write-Host "Today is $(Get-Date -Format 'dddd, MMMM dd yyyy')" -ForegroundColor Yellow
```

3. Run it from the terminal:

```powershell
# Navigate to the folder where you saved it
Set-Location C:\PSLessons

# Run it
.\hello.ps1
```

The `.\` prefix means "run this file from the current directory." You need it — PowerShell won't run a script by name alone for security reasons.

---

## What you learned

- Scripts are saved as `.ps1` files
- Execution policy controls what runs — `RemoteSigned` at `CurrentUser` scope is the standard dev setting
- Run scripts with `.\scriptname.ps1` from the containing directory
- `$(...)` inside a string executes an expression and inserts the result
