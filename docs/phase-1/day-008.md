# Day 008 — Filtering with Where-Object

## Overview

You rarely want everything. You want the running services, not all services. The failed logon events, not all events. The processes above a certain memory threshold, not every process on the system.

`Where-Object` is how you filter. It evaluates every object passing through the pipeline and only lets through the ones that match your condition.

---

## Basic filtering

```powershell
# Services that are currently running
Get-Service | Where-Object { $_.Status -eq "Running" }

# Processes using more than 100MB of memory
Get-Process | Where-Object { $_.WorkingSet -gt 100MB }

# Files larger than 1GB
Get-ChildItem C:\ -Recurse -File | Where-Object { $_.Length -gt 1GB }
```

---

## The shorthand syntax

PowerShell 3.0+ introduced a cleaner syntax for simple comparisons:

```powershell
# Classic syntax
Get-Service | Where-Object { $_.Status -eq "Running" }

# Shorthand syntax (same result)
Get-Service | Where-Object Status -eq "Running"
```

Use whichever is clearer. For complex conditions, use the classic block syntax.

---

## Multiple conditions

```powershell
# Services that are stopped AND start automatically
Get-Service | Where-Object { $_.Status -eq "Stopped" -and $_.StartType -eq "Automatic" }

# Processes named either chrome or firefox
Get-Process | Where-Object { $_.Name -eq "chrome" -or $_.Name -eq "firefox" }
```

---

## What you learned

- `Where-Object` filters pipeline objects based on a condition
- `$_` is the current object in the filter block
- Use `-and` / `-or` to combine conditions
- Shorthand syntax works for simple single-property comparisons
