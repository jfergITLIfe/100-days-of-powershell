# Day 009 — Selecting Properties with Select-Object

## Overview

PowerShell objects have a lot of properties. When you run `Get-Process`, each process object carries dozens of fields — most of which you don't need. `Select-Object` lets you pick exactly what you want and cut out the noise.

In real work — and especially in security work — clean, focused output is the difference between a useful report and a wall of data nobody reads.

---

## Selecting specific properties

```powershell
# Only show process name and memory usage
Get-Process | Select-Object Name, WorkingSet

# Show the first 10 items
Get-Process | Select-Object -First 10

# Show the last 5 items
Get-Process | Select-Object -Last 5

# Remove duplicates
Get-Process | Select-Object -Property Name -Unique
```

---

## Creating calculated properties

You can create new custom properties on the fly using a hashtable:

```powershell
Get-Process | Select-Object Name, @{
    Name = "MemoryMB"
    Expression = { [math]::Round($_.WorkingSet / 1MB, 2) }
}
```

This creates a new column called `MemoryMB` that converts bytes to megabytes. You'll use this pattern constantly when building reports.

---

## Expanding nested properties

```powershell
# Get just the values of a property (not wrapped in an object)
Get-Process | Select-Object -ExpandProperty Name
```

`-ExpandProperty` is useful when you want a plain list of values rather than a table.

---

## What you learned

- `Select-Object` controls which properties appear in your output
- `-First` and `-Last` limit result count
- `-Unique` deduplicates
- Calculated properties with `@{Name=...; Expression=...}` create custom columns
- `-ExpandProperty` gives you raw values instead of objects
