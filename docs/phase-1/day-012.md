# Day 012 — Arrays

## Overview

An array is an ordered list of items. In PowerShell, arrays are everywhere — the output of almost any cmdlet that returns multiple objects is an array. Knowing how to create them, access items, and work through them is foundational.

---

## Creating and accessing arrays

```powershell
# Create an array
$computers = "PC-001", "PC-002", "PC-003", "SERVER-01"

# Access by index (zero-based)
$computers[0]     # PC-001
$computers[-1]    # SERVER-01 (last item)
$computers[1..2]  # PC-002, PC-003 (range)

# Count items
$computers.Count

# Check if a value exists
$computers -contains "PC-002"    # True
```

---

## Adding to arrays

```powershell
# += adds an item (creates a new array internally)
$computers += "PC-004"

# ArrayList is better for frequent additions
$list = [System.Collections.ArrayList]@("PC-001", "PC-002")
$list.Add("PC-003")
$list.Remove("PC-001")
```

---

## Looping through arrays

```powershell
foreach ($computer in $computers) {
    Write-Host "Checking: $computer"
}

# Or with the pipeline
$computers | ForEach-Object { Write-Host "Checking: $_" }
```

---

## What you learned

- Arrays store ordered lists; indexes are zero-based
- `-1` accesses the last item; ranges with `..` slice a subset
- `-contains` checks membership
- `ArrayList` is more efficient for dynamic lists
