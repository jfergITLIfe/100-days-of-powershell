# Day 002 — Navigating the File System

## Overview

Before you can automate anything, you need to know where you are. PowerShell lets you move through the file system the same way you'd click through folders in Windows Explorer — except faster, scriptable, and without touching a mouse.

This is also the day you start building a habit that separates people who actually use PowerShell from people who just read about it: **always know your current location before you run a command.**

---

## Core commands

```powershell
# Where am I right now?
Get-Location

# List everything in the current folder
Get-ChildItem

# Move into a specific folder
Set-Location C:\Users\YourName\Desktop

# Go back up one level
Set-Location ..

# Go to your home directory
Set-Location ~
```

---

## Aliases — the shortcuts

PowerShell has short aliases that mirror Linux/Unix commands. They map directly to the full cmdlet names:

| Alias | Full cmdlet | What it does |
|-------|-------------|--------------|
| `pwd` | `Get-Location` | Print working directory |
| `ls` or `dir` | `Get-ChildItem` | List contents |
| `cd` | `Set-Location` | Change directory |

The aliases exist so experienced Linux/Unix users aren't completely lost on day one. In scripts, always use the full cmdlet names — it makes your code readable to anyone. In the terminal, use whatever's faster for you.

---

## Going deeper with Get-ChildItem

```powershell
# Show hidden files too
Get-ChildItem -Force

# Only show files (no folders)
Get-ChildItem -File

# Only show folders
Get-ChildItem -Directory

# Search recursively through all subfolders
Get-ChildItem -Recurse

# Find all .txt files in a folder
Get-ChildItem -Path C:\Logs -Filter *.txt
```

---

## What you learned

- `Get-Location` — shows your current path
- `Get-ChildItem` — lists files and folders
- `Set-Location` — moves you through the file system
- Aliases (`pwd`, `ls`, `cd`) exist and work
- `-Force`, `-File`, `-Directory`, `-Recurse` are useful flags
