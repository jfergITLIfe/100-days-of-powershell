# Day 014 — Reading User Input with Read-Host

## Overview

Scripts that only do one fixed thing are useful. Scripts that ask you what to do are tools. `Read-Host` is how you make your scripts interactive — prompting the user for input at runtime instead of hardcoding everything.

This is also the day you write your first script that actually feels like a real program.

---

## Basic usage

```powershell
# Prompt for input and store it
$name = Read-Host "Enter your name"
Write-Host "Hello, $name!"

# Prompt for a secure string (password — input is masked)
$password = Read-Host "Enter password" -AsSecureString
```

---

## A simple interactive script

```powershell
$target = Read-Host "Enter a computer name or IP to ping"
$result = Test-Connection -ComputerName $target -Count 2 -Quiet

if ($result) {
    Write-Host "$target is online." -ForegroundColor Green
} else {
    Write-Host "$target is not responding." -ForegroundColor Red
}
```

Save this as `ping-check.ps1` and run it. That's a real tool — one you could hand to a colleague.

---

## Adding color to output

```powershell
Write-Host "Success" -ForegroundColor Green
Write-Host "Warning" -ForegroundColor Yellow
Write-Host "Error"   -ForegroundColor Red
Write-Host "Info"    -ForegroundColor Cyan
```

Color makes terminal output dramatically more readable. Use it deliberately.

---

## What you learned

- `Read-Host` pauses execution and waits for user input
- `-AsSecureString` masks password input
- `Write-Host` supports `-ForegroundColor` for colored output
- Interactive scripts are more useful and more professional
