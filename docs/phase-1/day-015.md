# Day 015 — Formatting Output

## Overview

Raw PowerShell output is functional. Formatted output is professional. When you're writing scripts that get handed to a manager, dropped into a ticket, or shared with a team, the difference between a wall of text and clean aligned columns matters.

One important rule before we start: **formatting cmdlets should always be last in the pipeline.** Once you pipe to `Format-Table` or `Format-List`, you get a formatting object — not a data object. You can't pipe it further into `Export-Csv` or `Where-Object`. Format at the very end, after all your filtering and sorting.

---

## Format-Table

```powershell
# Default table output
Get-Service | Format-Table

# Choose specific columns
Get-Service | Format-Table Name, Status, StartType

# Auto-size columns to fit content
Get-Service | Format-Table Name, Status -AutoSize

# Wrap content instead of truncating
Get-Service | Format-Table Name, Status -Wrap
```

---

## Format-List

Shows each object as a vertical property list — better for objects with many properties:

```powershell
# All properties for one process
Get-Process -Name explorer | Format-List *

# Specific properties as a list
Get-Service -Name wuauserv | Format-List Name, Status, StartType, DisplayName
```

---

## Format-Wide

One column of values — useful for simple lists:

```powershell
Get-Process | Format-Wide Name -Column 4
```

---

## Out-GridView

Sends output to an interactive pop-up table — great for quick exploration:

```powershell
Get-Process | Out-GridView
Get-Service | Out-GridView -Title "Services on $env:COMPUTERNAME"
```

---

## What you learned

- `Format-Table` — columnar output, use `-AutoSize` for clean alignment
- `Format-List` — vertical property listing, great for detail views
- `Format-Wide` — single-value column layout
- `Out-GridView` — interactive pop-up grid for quick browsing
- Formatting goes **last** in the pipeline
