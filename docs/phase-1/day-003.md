# Day 003 — Variables

## Overview

Variables let you store information so you can use it later. In PowerShell, every variable starts with a `$`. That's the whole rule. No type declarations, no `var` or `let` — just assign and go.

This is one of those things that sounds simple but unlocks everything. Once you can store data in a variable, your scripts stop being one-liners and start being actual tools.

---

## Core syntax

```powershell
# Store a string
$name = "Jacob"

# Store a number
$age = 30

# Store the result of a command
$desktop = Get-Location

# Use the variable in output
Write-Host "My name is $name and I am $age years old."

# Check what type a variable is
$name.GetType()
$age.GetType()
```

---

## String interpolation

When you put a variable inside **double quotes**, PowerShell replaces it with its value. This is called interpolation — and it's one of the most useful things about the language.

```powershell
$city = "Houston"
Write-Host "I work in $city"       # Outputs: I work in Houston
Write-Host 'I work in $city'       # Outputs: I work in $city (single quotes = literal)
```

Single quotes mean "take this exactly as written." Double quotes mean "expand any variables you find." Know which one you need.

---

## What you learned

- Variables always start with `$`
- Assignment uses `=`
- Double quotes expand variables, single quotes don't
- `GetType()` reveals the data type of any variable
