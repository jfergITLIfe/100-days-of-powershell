# Day 017 — Comments & Code Readability

## Overview

You will read your own scripts six months from now and have no idea what you were thinking. Your colleagues will read your scripts and need to understand them fast. Comments and clean formatting aren't optional polish — they're how professional scripts get maintained, trusted, and reused.

This day is short. The habits you build here will pay off for the rest of your career.

---

## Single-line comments

```powershell
# This is a single-line comment
Get-Process  # You can also comment at the end of a line
```

---

## Multi-line (block) comments

```powershell
<#
    This is a block comment.
    Use it for longer explanations, 
    section headers, or disabling multiple lines.
#>
```

---

## Script header block

Every script you write for production should start with a header. Here's a clean template:

```powershell
<#
.SYNOPSIS
    Brief one-line description of what this script does.

.DESCRIPTION
    Longer explanation. What problem does it solve? Who runs it? When?

.PARAMETER ComputerName
    The target computer to run against.

.EXAMPLE
    .\Get-SystemInfo.ps1 -ComputerName "SERVER-01"

.NOTES
    Author:  Jacob Ferguson
    Date:    2024-01-01
    Version: 1.0
#>
```

This isn't just documentation — it's what `Get-Help` reads when someone runs `Get-Help .\YourScript.ps1`. Professional.

---

## Readability habits

```powershell
# Bad — everything crammed together
Get-Process|Where-Object{$_.WorkingSet -gt 100MB}|Select-Object Name,WorkingSet|Sort-Object WorkingSet -Descending

# Good — spaced and broken across lines
Get-Process |
    Where-Object { $_.WorkingSet -gt 100MB } |
    Select-Object Name, WorkingSet |
    Sort-Object WorkingSet -Descending
```

Same result. One is readable by a human under pressure. Use line continuation (`|` at end of line) freely.

---

## What you learned

- `#` for single-line comments, `<# #>` for blocks
- Comment-based help turns your script into self-documenting documentation
- Readable pipelines use line breaks after each `|`
- Your future self and your teammates will thank you
