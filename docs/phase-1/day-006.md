# Day 006 — Get-Help & Get-Command

## Overview

This is the day that separates people who memorize commands from people who can figure anything out. `Get-Help` and `Get-Command` are PowerShell's built-in documentation and discovery system. Learn these two cmdlets and you'll never be fully stuck again — even in an environment where you can't Google anything.

That's not hypothetical. If you work in a secured environment, an air-gapped network, or an incident response scenario, this is how you work.

---

## Get-Help

```powershell
# Basic help for a cmdlet
Get-Help Get-ChildItem

# Full detailed help
Get-Help Get-ChildItem -Full

# Just the examples (the most useful flag)
Get-Help Get-ChildItem -Examples

# Open help in a separate window
Get-Help Get-ChildItem -ShowWindow

# Update local help files (run once after install)
Update-Help
```

---

## Get-Command

```powershell
# Find all cmdlets with "process" in the name
Get-Command *process*

# Find all cmdlets that use the verb "Get"
Get-Command -Verb Get

# Find all cmdlets related to a noun
Get-Command -Noun Service

# Find commands from a specific module
Get-Command -Module ActiveDirectory
```

---

## The pattern to build

When you encounter a cmdlet you don't know:
1. `Get-Help <cmdlet> -Examples` — see it in action first
2. `Get-Help <cmdlet> -Full` — read the parameter details
3. Run it — learn by doing

When you need to do something but don't know the cmdlet:
1. `Get-Command -Verb <verb>` — what actions exist?
2. `Get-Command *<keyword>*` — search by keyword
3. `Get-Help about_<topic>` — conceptual help articles

---

## What you learned

- `Get-Help` retrieves documentation for any cmdlet
- `-Examples` is the most useful flag to start with
- `Get-Command` discovers cmdlets by name, verb, or noun
- These two tools make you self-sufficient at the terminal
