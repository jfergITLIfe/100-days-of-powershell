# Day 004 — Data Types

## Overview

PowerShell figures out data types automatically most of the time — but knowing what type you're working with matters the moment you try to do math on something that's secretly a string, or compare values that aren't what you think they are.

This day will save you hours of future debugging.

---

## The types you'll use most

```powershell
# String — text
$hostname = "DESKTOP-01"
$hostname.GetType().Name    # String

# Integer — whole number
$port = 443
$port.GetType().Name        # Int32

# Boolean — true or false
$isAdmin = $true
$isAdmin.GetType().Name     # Boolean

# Double — decimal number
$version = 7.4
$version.GetType().Name     # Double

# Null — nothing
$result = $null
$null -eq $result           # True
```

---

## Forcing a type

```powershell
# Cast a string to an integer
[int]$port = "443"
$port + 1    # 444

# Without the cast
$port = "443"
$port + 1    # Error — can't add a number to a string
```

---

## Why this matters in security work

When you're parsing log files, querying APIs, or reading CSV data, everything comes back as a string by default. If you want to do math on a port number or compare timestamps, you need to know how to cast types explicitly. Day 4 is where that awareness starts.

---

## What you learned

- PowerShell's main types: `String`, `Int32`, `Boolean`, `Double`, `null`
- `GetType().Name` reveals the type of any variable
- Square bracket casting `[int]`, `[string]`, `[bool]` forces a type conversion
- Type mismatches cause errors — knowing this early saves debugging time later
