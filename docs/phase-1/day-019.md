# Day 019 — PowerShell Profiles

## Overview

A PowerShell profile is a script that runs automatically every time you open a new PowerShell session. Think of it as your personal shell configuration — custom aliases, functions you use constantly, a welcome message, your prompt style. Once you set it up, every terminal you open is already tuned to how you work.

---

## Finding your profile

```powershell
# The path to your current user profile
$PROFILE

# Check if it exists yet
Test-Path $PROFILE
```

There are actually four profile locations (current user/all users × current host/all hosts). `$PROFILE` gives you the most common one: current user, current host.

---

## Creating and editing your profile

```powershell
# Create the file if it doesn't exist
if (-not (Test-Path $PROFILE)) {
    New-Item -Path $PROFILE -ItemType File -Force
}

# Open it in VS Code
code $PROFILE

# Or open in Notepad
notepad $PROFILE
```

---

## A starter profile worth using

```powershell
# ── Aliases ─────────────────────────────────────────
Set-Alias ll Get-ChildItem
Set-Alias grep Select-String

# ── Useful functions ─────────────────────────────────
function which ($command) {
    Get-Command $command | Select-Object -ExpandProperty Source
}

function uptime {
    (Get-Date) - (gcim Win32_OperatingSystem).LastBootUpTime
}

# ── Welcome message ──────────────────────────────────
Write-Host "Welcome back, $env:USERNAME" -ForegroundColor Cyan
Write-Host "$(Get-Date -Format 'dddd, MMMM dd')" -ForegroundColor DarkGray
```

---

## Reload your profile without restarting

```powershell
. $PROFILE
```

The `. ` (dot-source) operator runs a script in the current session scope. Any functions or variables it defines become available immediately.

---

## What you learned

- `$PROFILE` holds the path to your personal profile script
- Profiles run automatically on every new session
- You can define aliases, functions, and startup messages
- `. $PROFILE` reloads it without restarting the terminal
