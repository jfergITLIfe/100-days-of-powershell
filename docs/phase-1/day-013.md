# Day 013 — Hashtables

## Overview

A hashtable stores data as key-value pairs. Where an array says "give me item number 3," a hashtable says "give me the value for this key." Think of it like a lookup table — you define the label and the data that goes with it.

You'll use hashtables constantly: building custom objects, passing parameters via splatting, building config maps, storing lookup data. This is a critical data structure to get comfortable with.

---

## Creating and accessing hashtables

```powershell
# Create a hashtable
$server = @{
    Name     = "WEB-01"
    IP       = "10.0.0.10"
    OS       = "Windows Server 2022"
    IsOnline = $true
}

# Access by key
$server["Name"]       # WEB-01
$server.IP            # 10.0.0.10

# Add a new key
$server["Role"] = "Web Server"

# Remove a key
$server.Remove("IsOnline")

# Check if a key exists
$server.ContainsKey("OS")    # True

# List all keys
$server.Keys

# List all values
$server.Values
```

---

## Looping through a hashtable

```powershell
foreach ($key in $server.Keys) {
    Write-Host "$key = $($server[$key])"
}
```

---

## Ordered hashtables

By default, hashtables don't preserve insertion order. If order matters:

```powershell
$config = [ordered]@{
    Step1 = "Install"
    Step2 = "Configure"
    Step3 = "Verify"
}
```

---

## What you learned

- Hashtables store key-value pairs with `@{ Key = Value }`
- Access values with `$ht["Key"]` or `$ht.Key`
- `ContainsKey()`, `.Keys`, `.Values` are your main inspection tools
- `[ordered]` preserves insertion order
