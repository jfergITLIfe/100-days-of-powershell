# Day 020 — Phase 1 Capstone: System Info Reporter

Twenty days in. You've learned the shell, variables, data types, operators, the pipeline, filtering, formatting, strings, arrays, hashtables, user input, output streams, scripts, and profiles. Now you put it all together.

This is your first real deliverable. It goes on your GitHub. It proves you were here and you did the work.

---

## The project

Build a **System Info Reporter** — an interactive PowerShell script that collects key information about the local machine and outputs a clean, formatted report.

---

## Requirements checklist

Your script must do all of the following:

- [ ] Use a proper comment-based help header (`.SYNOPSIS`, `.DESCRIPTION`, `.NOTES`)
- [ ] Collect and display: hostname, current user, OS version, uptime, CPU name, total RAM (GB), disk usage on C:\ (used/total/free in GB)
- [ ] Collect and display: top 5 processes by memory usage (Name + MemoryMB)
- [ ] Collect and display: count of running services vs stopped services
- [ ] Format all output cleanly — use `Format-Table` or manual `Write-Host` with alignment
- [ ] Export the report to a `.txt` file named `SystemReport_<hostname>_<date>.txt`
- [ ] Print a colored summary line at the end confirming the export path

## Hints

- `Get-CimInstance Win32_OperatingSystem` has `.Caption`, `.LastBootUpTime`, `.TotalVisibleMemorySize`
- `Get-CimInstance Win32_Processor` has `.Name`
- `Get-PSDrive C` has `.Used` and `.Free` (in bytes — divide by 1GB)
- `(Get-Date) - $os.LastBootUpTime` gives you a `TimeSpan` object with `.Days`, `.Hours`, `.Minutes`
- Pipe your report sections through `Out-String` before writing to file, or build a string array and use `Set-Content`
