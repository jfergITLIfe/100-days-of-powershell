# Day 010 — Sorting & Grouping Output

## Overview

Ten days in. You can navigate the shell, store data, filter it, and shape it. Today you add the last two pipeline fundamentals: sorting and grouping. Together with `Where-Object` and `Select-Object`, these form the core pattern you'll use in almost every script you ever write.

---

## Sort-Object

```powershell
# Sort processes by name (ascending — default)
Get-Process | Sort-Object Name

# Sort by memory usage, largest first
Get-Process | Sort-Object WorkingSet -Descending

# Sort by multiple properties
Get-Service | Sort-Object Status, DisplayName
```

---

## Group-Object

`Group-Object` aggregates objects by a shared property value — like a GROUP BY in SQL.

```powershell
# Count services grouped by status
Get-Service | Group-Object Status

# Count processes grouped by name
Get-Process | Group-Object Name | Sort-Object Count -Descending
```

The output shows the group name, the count, and the objects in each group.

---

## Measure-Object

When you just want stats — count, sum, average, min, max:

```powershell
# How many processes are running?
Get-Process | Measure-Object

# Average memory across all processes
Get-Process | Measure-Object WorkingSet -Average

# Full stats on file sizes in a folder
Get-ChildItem C:\Windows -File | Measure-Object Length -Sum -Average -Maximum -Minimum
```

---

## Putting it all together

```powershell
# Top 5 processes by memory, clean output
Get-Process |
    Sort-Object WorkingSet -Descending |
    Select-Object -First 5 |
    Select-Object Name, @{Name="MemoryMB"; Expression={ [math]::Round($_.WorkingSet/1MB,1) }}
```

---

## What you learned

- `Sort-Object` sorts by any property, ascending or descending
- `Group-Object` aggregates and counts by a property
- `Measure-Object` gives you count, sum, average, min, and max
- Combining all the pipeline tools gives you SQL-like data analysis from the command line
