# Day 016 — Write-Output vs Write-Host

## Overview

This trips up beginners constantly — and honestly, a lot of experienced scripters get it wrong too. `Write-Output` and `Write-Host` both print things to the screen, but they do fundamentally different things under the hood. Getting this wrong means your scripts break in ways that are hard to debug.

---

## The core difference

```powershell
# Write-Host — prints directly to the console display
# Output cannot be captured or piped
Write-Host "This goes to the screen only"

# Write-Output — sends objects to the pipeline/output stream
# Output CAN be captured, piped, or redirected
Write-Output "This can be captured"
```

The test:

```powershell
$a = Write-Host "hello"    # $a is empty — Write-Host bypasses the pipeline
$b = Write-Output "hello"  # $b = "hello" — Write-Output goes into the stream
```

---

## When to use each

**Use `Write-Host` when:**
- You want status messages, progress indicators, or colored feedback
- The message is for the human watching — not for the script to consume

**Use `Write-Output` (or just return the object) when:**
- Your function/script needs to pass data to something else
- The output might be piped, captured in a variable, or exported

---

## The other output cmdlets

```powershell
Write-Verbose "Detailed diagnostic info"   # Only shows with -Verbose flag
Write-Warning "Something looks off"        # Yellow warning prefix
Write-Error "Something broke"              # Red error, writes to error stream
Write-Debug "Step 3 reached"              # Only shows with -Debug flag
```

These let you build scripts with multiple layers of feedback — clean output by default, detailed logging when you need it.

---

## What you learned

- `Write-Host` writes to the display only — cannot be piped or captured
- `Write-Output` writes to the output stream — can be piped and captured
- `Write-Verbose`, `Write-Warning`, `Write-Error`, `Write-Debug` each target different streams
- In functions, just returning an object (no Write cmdlet) is often the cleanest approach
