## What is this?

100 Days of PowerShell is a free, open-source, self-paced challenge that takes you from zero PowerShell experience to writing real defensive security scripts — in 100 days of 5–10 minute lessons.

No fluff. No theory dumps. Every day has one concept, working code, and a challenge that proves you did the work.

---

## The five phases

| Phase | Days | Focus |
|-------|------|-------|
| 1 — Foundations | 001–020 | Cmdlets, variables, pipeline, your first scripts |
| 2 — Scripting Core | 021–040 | Logic, loops, functions, error handling |
| 3 — Files, Systems & Data | 041–060 | File ops, registry, event logs, CSV/JSON |
| 4 — Automation & Real-World Tools | 061–080 | Modules, AD, Azure, REST APIs |
| 5 — Security Pipeline | 081–100 | Threat hunting, IR triage, Sentinel integration |

---

## How to take the challenge

1. **Fork this repo** — this becomes your personal challenge journal
2. **Work through each day** in order — each lesson is in `docs/phase-X/day-XXX.md`
3. **Do the daily challenge** — paste your Gist or screenshot in [Discussions → Daily Submissions](https://github.com/jfergitlife/100-days-of-powershell/discussions/categories/daily-submissions)
4. **Complete the capstones** — Days 20, 40, 60, 80, and 100 are full projects with a guided submission template
5. **Get on the showcase wall** — finish a capstone and submit a PR to add yourself to `showcases.json`
